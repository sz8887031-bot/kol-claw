## 二、初筛阶段（快速筛选）

### 2.1 筛选标准

**硬性指标：**
- ✅ 粉丝数：**1千 - 20万**（低粉爆款达人）⭐ 核心策略
- ✅ 互动率：> 3%（关键指标）
- ✅ 平均点赞：> 500
- ✅ 账号类型：个人账号（非企业号）
- ✅ 内容垂直度：与关键词相关

**核心理念：低粉高互动 = 高性价比**
- 1万粉丝，平均点赞 1000 → 互动率 10%（优质）
- 5万粉丝，平均点赞 500 → 互动率 1%（差）

**排除标准：**
- ❌ 粉丝数 < 1千（影响力太小）
- ❌ 粉丝数 > 50万（报价太高，不符合低成本策略）
- ❌ 互动率 < 1%（粉丝质量差）
- ❌ 企业号、品牌号
- ❌ 明显的广告号

### 2.2 快速筛选脚本

**使用现有数据快速筛选：**

```python
import json

with open('<MEDIACRAWLER_DIR>/discovered_creators.json', 'r') as f:
    data = json.load(f)

# 筛选 1千-20万粉丝的达人（低粉爆款策略）
qualified = [
    c for c in data['creators']
    if 1000 <= c.get('follower_count', 0) <= 200000
]

# 按粉丝数排序（从低到高，优先低粉达人）
qualified.sort(key=lambda x: x.get('follower_count', 0))

print(f"初筛通过: {len(qualified)}/{data['total_creators']}")
print("\n低粉达人（1千-5万）：")
low_fans = [c for c in qualified if c['follower_count'] < 50000]
print(f"数量: {len(low_fans)}")
for c in low_fans[:10]:
    print(f"- {c['creator_name']}: {c['follower_count']:,} 粉丝")

print("\n中粉达人（5万-20万）：")
mid_fans = [c for c in qualified if 50000 <= c['follower_count'] <= 200000]
print(f"数量: {len(mid_fans)}")
for c in mid_fans[:10]:
    print(f"- {c['creator_name']}: {c['follower_count']:,} 粉丝")
```

### 4.2 评级标准（v2.0）

**评级体系：**

| 评级 | 条件 | 说明 | 投放建议 |
|------|------|------|---------|
| **S级** | 粉赞比>5% + 平均点赞>1000 + 样本≥5条 + 稳定性>30% | 优质且稳定 | 强烈推荐 |
| **A级** | 粉赞比>5% + 平均点赞>1000，但样本<5条或不稳定 | 优质但有风险 | 高风险高回报 |
| **B级** | 粉赞比2-5% | 良好 | 可小额测试 |
| **C级** | 粉赞比1-2% | 一般 | 需谨慎 |
| **D级** | 粉赞比<1% | 差 | 不建议投放 |

**稳定性计算：**
```python
稳定性 = 最低点赞 / 平均点赞
```
- 稳定性 > 30%：数据稳定，风险低
- 稳定性 < 30%：数据波动大，可能遇到低谷期

**风险标签：**
- **新号(N条)**：视频数<5条，样本不足
- **不稳定(最低X赞)**：最低点赞<平均点赞30%
- **下降趋势**：近期数据明显下降

### 4.3 分类规则

**优质达人.csv**：S级 + A级
- S级：优质且稳定，可直接投放
- A级：优质但有风险，根据风险偏好选择

**备选达人.csv**：B/C/D级
- B级：可小额测试
- C/D级：观察或放弃

### 4.4 CSV字段

```
达人名称,粉丝数,点赞1(最新),点赞2,点赞3,点赞4,点赞5(最早),
平均点赞,粉赞比(%),评级,趋势,风险标签,样本量,内容定位,主页链接,投放评估
```

**关键字段说明：**
- **评级**：S/A/B/C/D
- **风险标签**：无、新号(N条)、不稳定(最低X赞)、下降趋势
- **样本量**：实际视频数（2-5条）
- **投放评估**：基于评级、稳定性、样本量的综合评估

## 五、质量评级阶段

### 5.1 自动评级脚本

```python
def rate_creator(creator_data):
    """自动评级函数"""
    videos = creator_data['top_5_videos']
    follower_count = creator_data.get('follower_count', 0)

    # 计算平均数据
    avg_likes = sum(v['like_count'] for v in videos) / len(videos)
    avg_comments = sum(v['comment_count'] for v in videos) / len(videos)
    avg_shares = sum(v['share_count'] for v in videos) / len(videos)
    avg_collects = sum(v['collect_count'] for v in videos) / len(videos)

    # 互动率
    engagement_rate = (avg_likes / follower_count * 100) if follower_count > 0 else 0

    # 互动指数
    engagement_score = avg_likes + avg_comments * 10 + avg_shares * 5 + avg_collects * 3

    # 评级
    if engagement_rate > 50 and avg_likes > 10000:
        return "⭐⭐⭐⭐⭐", "强烈推荐"
    elif engagement_rate > 10 and avg_likes > 5000:
        return "⭐⭐⭐⭐", "推荐"
    elif engagement_rate > 5 and avg_likes > 1000:
        return "⭐⭐⭐", "谨慎考虑"
    elif engagement_rate > 1:
        return "⭐⭐", "不推荐"
    else:
        return "⭐", "放弃"
```

### 12.2 评估标准更新（v2.0）

**多维度评估标准（2026-03-04 更新）：**

**核心理念：稳定性 > 粉赞比**

不能只看粉赞比，还要看数据稳定性。高粉赞比但数据不稳定的达人，投放时可能遇到低谷期，ROI会爆炸。

**示例：**
- 兔哥来报：粉赞比70%，但5条视频点赞为 76、732、10231、15274、551
- 问题：最低只有76赞，如果投放遇到低谷期，7千粉要几千块，只有76赞，ROI灾难
- 结论：标记为A级+不稳定，供用户根据风险偏好选择

**评级逻辑：**

| 评级 | 粉赞比 | 平均点赞 | 稳定性 | 样本量 | 说明 |
|------|--------|---------|--------|--------|------|
| S级 | >5% | >1000 | >30% | ≥5条 | 优质且稳定，可直接投放 |
| A级 | >5% | >1000 | <30% 或 <5条 | 2-5条 | 优质但有风险，高风险高回报 |
| B级 | 2-5% | - | - | 2-5条 | 良好，可小额测试 |
| C级 | 1-2% | - | - | 2-5条 | 一般，需谨慎 |
| D级 | <1% | - | - | 2-5条 | 差，不建议投放 |

**新号处理：**
- 视频数<5条的达人不再跳过
- 标记为"新号(N条)"
- 放入备选达人.csv供用户手动筛选
- 示例：小宇同学-，只有3条视频，粉赞比590%，标记为B级+新号(3条)

**分类规则：**
- **优质达人.csv**：S级 + A级（高潜力，用户根据风险偏好选择）
- **备选达人.csv**：B/C/D级（可测试或观察）

### 12.3 分析脚本错误

**问题：** `analyze_creators.py` 报错 `KeyError: 'engagement_rate'`

**原因：** 脚本版本不一致，缺少某些字段

**解决方案：**
- 检查脚本是否为最新版本
- 确保 `creator_detail_analysis.json` 格式正确
- 如报错，手动检查缺失字段

### 12.4 搜索结果质量差

**问题：** 搜索到的达人质量不高，全部被标记为"放弃"

**原因分析：**
1. 关键词选择不当（低互动关键词）
2. 该领域达人整体质量差
3. 粉丝数范围设置不合理

**解决方案：**
1. **立即停止当前关键词**
2. **切换到高互动关键词**：
   - 自媒体运营类（小红书运营、抖音运营）
   - 办公软件类（Excel技巧、PPT制作）
   - 副业项目类（副业项目、副业赚钱）
3. **调整粉丝数范围**：
   - 如果都是大V，降低上限（5-20万）
   - 如果都是小号，提高下限（5千-50万）

### 12.5 工作流程不规范

**问题：** 跳过某些步骤，或不按顺序执行

**标准执行顺序（强制）：**

```
1. 环境检查 → check_env.sh
   ↓
2. 查看关键词库 → 关键词分类库.md
   ↓
3. 选择高互动关键词 → 修改 discover_creators.py
   ↓
4. 关键词搜索 → uv run python discover_creators.py
   ↓
5. 自动分析 → python3 analyze_creators.py
   ↓
6. 查看报告 → 达人质量分析报告_*.md
   ↓
7. 人工决策 → 选择核心推荐达人
```

**禁止跳过的步骤：**
- ❌ 不查看关键词库就选关键词
- ❌ 不运行 analyze_creators.py 就手动评估
- ❌ 不看最新评估规则就给出推荐

## 十三、飞书同步流程（2026-03-10新增）

### 13.1 同步时机

**何时需要同步到飞书：**
1. ✅ 新增确定合作的达人
2. ✅ 更新达人的建联状态
3. ✅ 修改达人的报价或级别
4. ✅ 更新达人的投放状态

**同步频率：**
- 每次更新"达人跟进表.csv"后立即同步
- 建议：每天结束工作前统一同步一次

### 13.2 同步命令

**标准流程（两步）：**

```bash
# 步骤1：提取确定合作达人到本地CSV
cd <PROJECT_DIR>/data
python3 sync_confirmed_creators.py

# 步骤2：同步到飞书
cd <PROJECT_DIR>
python3 sync_to_feishu_incremental.py
```

**一键同步（推荐）：**

```bash
# 创建快捷命令（首次执行）
echo "alias sync-kol='cd <PROJECT_DIR>/data && python3 sync_confirmed_creators.py && cd .. && python3 sync_to_feishu_incremental.py'" >> ~/.zshrc
source ~/.zshrc

# 使用
sync-kol
```

```bash
# 链接：https://bytedance.larkoffice.com/base/your_app_token_here
```
